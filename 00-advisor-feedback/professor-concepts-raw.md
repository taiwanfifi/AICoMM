「而且這個東西其實需要有一點 Support。你剛才在講說，那個以前……譬如說可以很複雜，所以你如果去看大家現在在講 Semantic Communication（語意通訊），因為以前其實就是一般的 Communication，那所以他們後來會講 Task-oriented 或是 Goal-oriented（目標導向）。
就是說，我今天在做這些傳輸——因為我們不再講一般的通訊，會遇到一個損掉（Loss）——可是我們很多的服務都是 End-to-End。所以在講說你那裡這個 End-to-End，我從這個 Task、我從工作的目標來看，我應該要傳什麼？要怎麼傳？怎麼樣才會好？我覺得大家在談是在談這一種。
那在這個情形之下，那應該要怎麼做？那相信 MCP 有它扮演的角色，可是絕對不是『應用』啦。我覺得……我在這個裡面我看不到（研究價值）。
因為我覺得未來這個 Agent Communication 其實會很重要。Agent Communication 就是……他們其實都是透過……未來很多人說有可能傳的不是 Packet（封包），是 Token 啦。因為我們現在在 Process 很多都是在處理 Token，所以未來有可能不是我們現在傳的這樣。
那只是說怎麼做？現在大家搞不清楚。因為你……我們以前在傳這個東西，比方說你說 Machine 或是說我們現在是『人』，人在傳跟機器在傳其實不一樣。可是未來我們其實大概都有 Agent，Agent 會幫我們——就是它也許會去協調很多不同種。然後問題是接下來要怎麼去運作，其實我也沒那麼知道。這個其實是大家現在談最多的。
就是說，未來有可能是 Agent Communication，不是像我們現在這樣。就是我們要去看一看這個每個東西要怎麼樣設計……那因為你談論到這個 Agent，那就牽扯到這個 Agent 大概是要怎麼運作？那 Agent 跟 Agent 之間，他們會產生什麼樣的行為？他們會出現什麼東西？這個大家現在其實是關心的是這種問題啦。」
