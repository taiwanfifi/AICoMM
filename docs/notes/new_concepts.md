不用微調，有讓LLM讀一次書就記住的新方法？！

傳統微調既慢又吃資源。

Sakana AI提出的Doc-to-LoRA（D2L）換了個思路：訓練一個小型Hypernetwork，之後面對任何新文件只要跑一次前向傳播，就能產生一組專屬的LoRA。之後LLM只要掛上 LoRA就像讀過並記住了那份文件，之後回答問題完全不需要再看原文，省下大量記憶體和時間。

處理長文時，D2L把長文Chunking成多段，每段各自產生 LoRA 再拼接，讓模型能消化遠超上下文窗口的內容。實驗中只用 256 token 短文訓練，卻在 32K+ token 測試中維持近乎完美的準確率，等於把 8K 上下文能力擴展了四倍。推理額外記憶體從 12GB 驟降到不到 50MB。

真實問答任務同樣亮眼：D2L只需0.55秒完成知識內化，傳統方法要72秒以上；記憶體從近80GB降到不到4GB，準確率還更高。跨模態實驗中，用VLM當編碼器，從沒見過圖片的純文字LLM 竟達到75%圖片分類正確率。

目前限制是訓練Hypernetwork仍需不少算力，且每換目標模型要重新訓練。但比起微調 D2L 展示了更高效的解方。

未來的 LLM 運算瓶頸將從 GPU 轉移到儲存 I/O ？！

Agent 場景中，模型與環境互動數十至數百輪，每輪只新增少量 token，但累積上下文極長，超過 95% 可直接從 KV-Cache 讀取。這讓瓶頸從傳統的 GPU 運算轉移到儲存 I/O，GPU 大部分時間不是在算，而是在等 KV-Cache 從遠端儲存讀回來。

現有 PD 分離架構中，只有 Prefill 引擎負責從儲存讀取 KV-Cache，其網卡頻寬被打滿，Decode 端的網卡卻閒著，進一步加劇了儲存 I/O 瓶頸。

DeepSeek 提出 DualPath：讓 Decode 端也能幫忙讀取 KV-Cache，再透過高速 RDMA 網路轉給 Prefill 端，把原本閒置的頻寬利用起來。搭配 QoS 流量隔離確保不影響模型推論、以及自適應排程平衡各節點負載。

在真實 Agentic 工作負載下，離線推論最高加速 1.87 倍，線上服務平均加速 1.96 倍，並展示至 1,152 GPU 的近線性擴展能力。
