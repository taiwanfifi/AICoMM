  P3：SatFM — 衛星通道 Foundation Model

  背景（教授版）

  我們建了一個 88M參數的 Masked Autoencoder，用3GPP
  NTN-TDL標準模型產生的22.4萬筆LEO衛星通道矩陣做自監督預訓練。概念類似
   BERT 學語言一樣：把通道矩陣遮住一部分，讓模型學會把它補回來。預訓練

  2. Data efficiency曲線不完整：只有1%和100%兩個點，中間的5%/10%/25%/5
  0%還沒跑完

  P5：Causal AI 無線資源管理

  進度：80/100 | 🟢 最穩定，接近可投稿

  背景（教授版）

  傳統機器學習在無線通訊中的大問題：訓練時的資料分佈和實際部署時不同（
  天氣變了、用戶密度變了、散射體變了），模型就崩了。我們用
  Pearl的因果推論框架三層級來解決：
  - Level 1（因果發現）：用GES等演算法從數據中找出哪些變數之間有因果關
  係（例如：用戶角度→波束選擇，vs 用戶數量→只是統計相關）
  - Level 2（因果特徵選擇）：只用因果父節點做預測，排除虛假相關性 →
  OOD環境下不崩
  - Level 3（反事實增強）：「如果用戶位置不同，但其他條件一樣，波束應
  該選幾號？」用反事實數據增強訓練

  兩個應用場景：(1) mmWave波束預測、(2) 多設備通道存取

  - 3個通道模型驗證：3GPP single-tap、DeepMIMO ray-tracing、3GPP
  CDL-C/D
  - 所有數值claims已與source JSON交叉驗證，零差異

  核心發現

  - GES+domain priors在single-tap和CDL上都robust，LASSO在CDL上也robust

  P6：SNN脈衝神經網路 OFDM 接收機

  進度：25/100 | 🔴 核心論述崩潰

  背景（教授版）

  傳統深度學習接收機用的是連續值運算（multiply-accumulate,
  MAC），每次乘法要消耗4.6 pJ（45nm CMOS）。脈衝神經網路（SNN）用二值
  脈衝傳遞信號，每次突觸運算（SynOp）只要0.9 pJ →
  理論上省5倍。我們把SNN接收機從之前文獻的16-QAM 擴展到64-QAM和256-QAM
  ，用可學習的LIF神經元參數（衰減常數、閾值）來適應更複雜的調變。

  目前做到哪裡

  - 140個結果JSON檔（results/ + results_r10/）
  - BER結果：SNN-Conv比ANN-Conv好2-6%
  - Paper 722行，8頁IEEE格式

  為什麼說「核心論述崩潰」

  三個independent reviewer（GPT-5.2=57.8, Gemini=54.5,
  Claude=53.1）一致指出同樣的致命問題：

  1. 能量故事是假的：SNN-Conv的能量有99.4%是MAC運算，只有0.6%是spike-d
  riven SynOp。所謂的「35%能量節省」完全來自網路架構比較小（更少的Conv
  層），不是因為「脈衝」本身。任何一個同樣大小的ANN都能達到類似節省。
  2. Baseline不公平：SNN-Conv用kernel (5,5,5,1) =
  45K參數，ANN-Conv用kernel (5,5,3,1) = 36K參數 →
  SNN多25%參數和更大的receptive field（17 vs 13
  subcarrier）。BER的改善可能純粹是因為模型更大。Fair
  baseline（matched architecture）的實驗因code bug（set_seed
  NameError）crash了，至今沒補跑。
  3. 測試環境不切實際：只用4個pilot（48個子載波中），64-QAM在15dB
  SNR的BER高達17%——這在實際系統中根本不能用，任何FEC都解不了。用8個以
  上pilot時，SNN的優勢就消失了。

  可能的轉向

  Gemini提出了一個非常有創意的方向：不要把SNN的16個timestep用在靜態數
  據上（人為的時間循環），而是讓SNN跨OFDM symbol累積膜電位 →
  用LIF膜電位當零成本的temporal
  memory來追蹤快衰落通道。在高Doppler場景下（如300 Hz），classical
  LS/MMSE在pilot之間無法追蹤通道變化，LSTM太貴，而SNN的膜電位天然就是t
  emporal state。這會把SNN從「便宜版的Conv1D」變成「解決真實無線問題的
  unique方法」。

  建議

  要嘛跑fair baseline修現有問題，要嘛直接pivot到Gemini建議的channel
  tracking方向

  P7：Split Diffusion 邊緣語意通訊

  進度：75/100 | 🟢 實驗全完成，最接近投稿

  DIM是確定性的，在任何步驟切割都不會損失品質（zero loss）。

  隨k線性下降，斜率和SNR有關。一個DRL（DQN）agent根據通道品質、用戶數
  量、設備算力等，動態決定最佳切割點k和是否使用ControlNet。

  P2：Agentic AI for Autonomous Network Management

  背景：別人做了什麼？

  2024–2025 年，Meta 發表 Confucius（SIGCOMM 2025）用 LLM
  做網路故障診斷，Microsoft 發表 MeshAgent（SIGMETRICS 2026）做多步驟
  推理網路修復。但這些工作都有一個共同問題：它們只跟簡單的 rule-based
  heuristic 比較，沒有跟數學最佳化（LP）比過。所以沒人知道 LLM
  到底是「真的比較聰明」，還是只是「比寫死的 if-else 好一點」。

  我們做了什麼？方法是什麼？

  我們建了一個 4-agent pipeline（Diagnoser → Planner → Executor →
  Validator），在一個可控的網路模擬器上測試。模擬器用 M/M/1
  排隊論計算延遲，支援 4 種場景：
  1. Predictive Scaling（流量預測）— 結構化問題
  2. Multi-Failure Triage（多故障排除）— 非結構化問題
  3. Capacity Planning（容量規劃）
  4. Mixed Scenario（混合場景）

  關鍵設計：我們加入了 LP 最佳化
  baseline（用線性規劃求解最佳路由），以及 GreedyGraph
  heuristic，公平地跟 LLM 比較。網路規模從 15 → 25 → 35 → 50 → 100
  節點逐步放大。

  具體數值結果

  R21 最終成績（10-seed 平均 ± 標準差）：

  ┌──────────────────────┬─────────┬───────────────┬──────────────┐

  核心發現：
  - 在「結構化」問題（如流量預測），LP 永遠贏 LLM —
  這是「負面結果」，但非常有價值
  - 在「非結構化」問題（多故障），LLM 大幅領先 heuristic +26.7pp（15
  nodes）、+43.3pp（50 nodes）
  - Alert confound 分解：我們發現 LLM 的表現有 11pp 來自「讀文字理解
  alert」，另外 13pp 來自「真正的拓撲推理」。移除 alert 後 LLM 仍達
  87.2%，證明它真的在做空間推理

  **規模效應：**Heuristic 從 76.3%（15 nodes）崩潰到 20.2%（50
  nodes），跌幅 -56.1pp。LLM 只跌 -15.4pp。這證明 LLM 的推理能力隨規模
   degrades gracefully。

  **統計嚴謹度：**2,614 次實驗，30-seed flagship，bootstrap CI +
  permutation test + Cohen's d 效應量。

  結論與可發表度：78/100

  論文已完稿（1,484 行 LaTeX），Gemini 給 89.3
  分。最大弱點是模擬器太簡化（M/M/1 而非 ns-3），以及 100-node
  場景需要更多 seed。建議投 IEEE JSAC SI on AI-Native Networks 或 IEEE
   Network Magazine。

  P3：SatFM — Foundation Model for LEO Satellite Channels

  背景：別人做了什麼？

  2024 年，LWM（Large Wireless Model）首次證明可以用 Masked
  Autoencoder 在地面通道矩陣上做 self-supervised pre-training，然後
  fine-tune 到下游任務（去噪、壓縮）。WiFo（2025）把這個概念擴展到時空
  頻域。但是，沒有人在衛星（NTN）通道上做過 Foundation Model —
  衛星通道有獨特的 Doppler shift（可達 ±24
  kHz）、極高仰角變化（10°–90°）、以及 Rician fading 特性。

  我們做了什麼？方法是什麼？

  SatFM 是一個 96M 參數的 Vision Transformer (ViT) Masked
  Autoencoder：
  1. Pre-training：在 224,000 張 3GPP NTN-TDL 通道矩陣上做 Masked
  Channel Modeling（隨機遮蔽 75% 的 patches，讓模型重建）
  2. Architecture：32×64 通道矩陣 → 切成 4×4 patches → ViT encoder (12
   layers) → decoder (4 layers)
  3. 下游任務：去噪（Denoising）、時間預測（Temporal Prediction）、CSI
   壓縮

  P4：World Models for Proactive 6G BS Sleep Management

  背景：別人做了什麼？

  5G/6G 綠色網路的核心問題：基站在低流量時浪費大量能源（佔電信業碳排
  70%+）。傳統方法用 PPO/DQN 等 model-free RL 做 reactive
  休眠控制，但遇到突發流量（traffic
  surge）時，因為基站需要喚醒延遲（wake-up
  latency），反應總是太慢。Hafner 等人（2023）提出 DreamerV3 world
  model 在遊戲 AI 領域展示了「先想像未來、再做決策」的 proactive
  planning 能力，但從未應用到電信領域。

  我們做了什麼？方法是什麼？

  搭建了一個 7-cell 六角蜂巢環境，每 15 分鐘一步，模擬 24 小時流量（含
   Poisson 突發）：
  1. World Model (RSSM)：學習環境的隱藏狀態轉移 $s_{t+1} = f(s_t,
  a_t)$
  2. MPC (CEM)：用 world model「想像」未來 H=8 步，用 Cross-Entropy
  Method 找最佳動作序列
  3. Wake-up latency：$K=3$ 步，基站從休眠到開機需要 3 步延遲
  4. Augmented Lagrangian reward：用拉格朗日乘數自動平衡能耗與 QoS
  約束

  P5：Causal AI for OOD-Robust Wireless Resource Management

  背景：別人做了什麼？

  所有 ML-for-wireless
  模型都有一個致命弱點：分佈外（OOD）崩潰。例如，用晴天數據訓練的 beam
   management DNN，一下雨就全崩（accuracy 從 90% 跌到 <50%）。原因是
  DNN 學到了「虛假相關」（spurious correlation）— 比如「下午 3
  點用戶多 → beam 5」，但這只是巧合。2024 年 IRM（Invariant Risk
  Minimization）和 GroupDRO 試圖解決 OOD，但沒有從因果層面根本解決。

  我們做了什麼？方法是什麼？

  我們把 Pearl 的因果推論三層級（Three-Level Causal
  Hierarchy）完整應用到無線網路：

  Level 1 — 因果發現（Discovery）： 用 GES 演算法 + 3GPP
  物理先驗（e.g., 「天氣不能被 beam 影響」）從數據中學出因果圖（DAG）

  Level 2 — 因果特徵選擇： 從 DAG 中只選「beam 的因果父節點」（如
  angle_az, distance），丟掉虛假特徵（如 n_ue_cell）

  Level 3 — 反事實估計： 用 SCM 模型回答「如果當時改用 beam
  3，throughput 會是多少？」

  P6：Spiking Neural Network OFDM Receiver

  背景：別人做了什麼？

  傳統 OFDM 接收器用 DNN（ANN）做通道估計和均衡，但 DNN 用的
  MAC（乘加累計）運算耗能大（~4.6 pJ/op in 45nm）。Spiking Neural
  Network（SNN）用脈衝（spike）傳遞資訊，每次突觸操作（SynOp）只需
  ~0.9 pJ — 理論上節能 5.1 倍。Gupta 等人（2024, 2025）已經用 SNN 做了
   16-QAM OFDM 接收器。我們的目標是擴展到更高階：64-QAM 和 256-QAM。

  我們做了什麼？方法是什麼？

  SNN-Conv 架構： LS 粗估通道 → ZF 均衡 → Conv1D 特徵萃取 → LIF (Leaky
   Integrate-and-Fire) 脈衝層（16 timesteps）→ 軟判決輸出

  LIF 神經元的 decay (β) 和 threshold (θ) 是 learnable 參數，用
  surrogate gradient（直通估計器）訓練。

  具體數值結果

  ├─────────┼──────────────┼──────────┼──────────┼───────┤
  │ QPSK    │    0.028     │  0.019   │  0.021   │ 0.027 │

  1. Baseline 不公平（Fatal）： ANN-Conv 用 kernel (5,5,3,1)，SNN-Conv

  topology），但 腳本 crash 後從未重跑。
  2. BER 太高（Fatal）： 64-QAM 在 15 dB 的 BER = 0.174（17.4%），任何

  P7：Adaptive Split Diffusion for Edge Semantic Communication

  背景：別人做了什麼？

  語義通訊（Semantic Communication）的目標是只傳「意義」而非原始
  bits。DiffJSCC（2024）首次把 diffusion model
  用在語義通訊上，在發送端壓縮圖像語義特徵，在接收端用 diffusion
  denoising 重建。但 DiffJSCC 有一個未解決的問題：denoising
  步數固定。在好通道（高 SNR）浪費算力，在差通道（低
  SNR）重建品質不夠。

  我們做了什麼？方法是什麼？

  Split Diffusion： 把 diffusion denoising 的 $T$ 步切成兩段：
  - 前 $k$ 步在 edge server（靠近基站，GPU 強但需要上傳）
  - 後 $T-k$ 步在 mobile device（本地，省頻寬但算力弱）

  關鍵發現 — DDIM 確定性不變性： 因為 DDIM（Denoising Diffusion
  Implicit Models）是確定性的（$\eta=0$），不管在哪裡切、在哪台機器上
  跑，重建結果完全相同（pixel-level deviation =
  0）。這是一個非常優雅的理論性質：split point
  只影響延遲和頻寬，不影響品質。

  DRL 自適應控制： 用 DQN 根據即時 SNR 和 edge server 負載動態選擇最佳
   split point $k^*$。延遲用 M/D/1 排隊模型建模。

  ControlNet 移除作為降級旋鈕： 在 edge 端移除 ControlNet module
  可以降低計算量（換取品質下降），作為 quality-latency 的 tradeoff
  機制。
